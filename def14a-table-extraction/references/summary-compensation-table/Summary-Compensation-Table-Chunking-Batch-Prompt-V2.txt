Split a filing-list CSV into smaller work chunks for the V3 Summary Compensation Table creation workflow.

Goal:
- Take `[SOURCE_CSV]`
- Split it into chunk CSVs of about 10 filings each
- Keep one filing per row from the source list
- Prepare a batch workspace for Summary Compensation Table processing

Instructions:
- Read `[SOURCE_CSV]`
- Treat each row as one filing work item
- Preserve the original row order from the source file
- Create chunk files with up to 10 filings per chunk
- Do not run extraction or QA during chunking
- Do not create Python scripts or helper code files in the workspace unless I explicitly ask
- If temporary code is absolutely necessary, use inline commands or `/tmp` and do not leave files in the project folder

Output structure:
- Create a batch-specific folder under `[OUTPUT_ROOT]`
- Inside that folder create:
  - `chunks/`
  - `merged/`
  - `chunk_manifest.csv`
  - `wave_manifest.csv`
  - `README.md`

Recommended output folder:
- `[OUTPUT_ROOT]/summary-compensation-table-v3`

Chunk naming:
- `failed-no-comp-v2_chunk-01.csv`
- `failed-no-comp-v2_chunk-02.csv`
- and so on

Manifest requirements:
- Create `chunk_manifest.csv` with columns:
  - `Chunk File`
  - `Filings In Chunk`
  - `First CIK`
  - `First Company`
  - `Last CIK`
  - `Last Company`

README requirements:
- Explain that each chunk should be processed with `references/summary-compensation-table/Summary-Compensation-Table-Agent-Plan-V3.md`
- Explain that agents should use only the provided filing URL and should not check local raw filing folders
- Explain that each chunk should produce:
  - `[chunk_name]_compensation_table_master.csv`
  - `[chunk_name]_report.csv`
- Explain that merge behavior is handled after all chunk threads are complete

Validation requirements:
- Confirm the number of filings in the source CSV
- Confirm the number of chunk files created
- Confirm that every source filing appears in exactly one chunk
- Confirm that all chunk CSVs parse cleanly

After completion, report:
- total filings
- number of chunks created
- output folder path
- chunk manifest path
